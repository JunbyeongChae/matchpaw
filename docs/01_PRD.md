## 0. 프로젝트 개요

이 문서는 **matchpaw** 의 제품 요구사항을 정의한다. matchpaw는 처음 유기동물 입양을 고려하는 사람이 라이프스타일 설문 5문항만으로 나에게 맞는 유기동물을 AI 기반으로 추천받고, 입양 이후 준비 체크리스트까지 안내받을 수 있는 플랫폼이다. 이 PRD를 기반으로 기능 구현 및 기술 결정이 이루어진다.

---

## 1. 배경 및 목표

### 1-1. 배경

국내 연간 유기동물 발생은 약 11만 3천 마리에 달하지만 입양률은 31%에 그친다(농림축산식품부, 2023). 입양을 원하는 사람은 증가하고 있지만 기존 플랫폼(APMS 등)은 지역·축종 필터만 제공해, "내 생활 패턴에 맞는 동물이 뭔지 모르겠다", "입양 후 어떻게 해야 하는지 아무도 안 알려준다"는 첫 입양자의 핵심 Pain Point를 해결하지 못한다. 첫 입양자의 진입 장벽을 낮추면 유기동물 입양률 제고로 이어진다는 판단 아래 이 프로젝트를 시작했다.

### 1-2. 목표

| # | 목표 | 연결 성공 기준 (→ 섹션 2) |
|---|---|---|
| G-01 | 설문 5문항으로 맞춤형 유기동물 추천 경험을 3단계 이내로 제공한다 | 매칭 플로우 완료 단계 수 |
| G-02 | AI 입양 체크리스트 자동 생성으로 준비 불안을 해소한다 | 체크리스트 자동 생성 정상 동작 |
| G-03 | 보호중 상태 실시간 반영으로 이미 입양된 동물 노출을 방지한다 | 보호중 상태 필터 정확도 |

### 1-3. 비목표

- 지도 기반 보호소 뷰 — 지도 API 비용·복잡도가 4주 개인 프로젝트 MVP 범위를 초과하므로 제외
- 커뮤니티 기능(게시판, 댓글) — 핵심 가치(매칭·체크리스트)와 무관, 추후 v2에서 고려
- 산책 알림 기능 — 기상청 API 연동 포함, 이번 개인 프로젝트 범위 밖
- 실제 결제 및 입양 신청 처리 — 보호소별 절차가 상이, 연락처 원클릭 연결로 대체
- 소셜 로그인 (OAuth) — MVP에서 이메일 인증으로 단순 시작, 추후 확장 가능

| AI 응답 시간 | Claude API 응답 5초 이내 (타임아웃 처리 포함) | 구현 완료 후 직접 테스트 | G-02 |
| 보호중 상태 필터 정확도 | 공공 API "보호중" 상태 동물만 목록 노출 | 구현 완료 후 직접 테스트 | G-03 |
| 모바일 정상 렌더링 | 375px 이상 전 구간 정상 렌더링 | 개발 완료 시점 | — |

### 출시 전 체크리스트

- [ ] 설문 5문항 입력 → AI 매칭 결과 카드와 감성 코멘트가 정상 렌더링된다
- [ ] AI 입양 체크리스트가 동물 정보 기반으로 자동 생성된다
- [ ] 유기동물 목록에 보호중 상태 동물만 표시된다
- [ ] 비회원은 일 2회 매칭 후 회원가입 유도 메시지가 표시된다
- [ ] 회원 찜 목록 저장·삭제가 정상 동작한다
- [ ] 최대 3마리 비교 테이블이 정상 동작한다
- [ ] 빈 상태·에러 상태·로딩 상태에서 안내 UI가 표시된다
- [ ] 모바일 375px 이상에서 레이아웃이 깨지지 않는다
- [ ] Chrome, Safari 최신 버전에서 정상 동작한다

## 3. 대상 사용자

### Primary — 첫 입양 고려 1인 가구 직장인

| 항목 | 내용 |
|---|---|
| 대표 유형 | 28세 1인 가구 직장인 (가명: 이지은) |
| 주요 특징 | 주 5일 출근·저녁 귀가·주말 활동적. 스마트폰 중심 정보 탐색. 반려동물 경험 없음 |
| Pain Point | "내 생활 패턴에 맞는 동물이 뭔지 모르겠다", "입양 후 어떻게 해야 하는지 아무도 안 알려준다" |
| 핵심 목표 | 막막함 없이 나에게 맞는 유기동물을 찾고, 입양 첫 날부터 자신 있게 시작하는 것 |

### Secondary — 재입양 희망자 또는 정보 탐색형 사용자

| 항목 | 내용 |
|---|---|
| 대표 유형 | 반려동물 경험 있는 30대, 추가 입양을 위해 유기동물 목록 탐색 중 |
| 주요 특징 | 이미 반려동물 지식 보유. 종류·지역 필터로 원하는 동물을 빠르게 찾고자 함 |
| Pain Point | 실시간 보호 상태가 반영 안 된 플랫폼에서 이미 입양된 동물을 보고 헛걸음한 경험 |
| 핵심 목표 | 지역·나이·종류 필터로 신뢰할 수 있는 데이터 기반으로 원하는 동물을 빠르게 찾는 것 |

## 4. 기능 요구사항

|---|---|---|

### 4-2. 라이프스타일 설문 (`/survey`)

|---|---|---|
| 5문항 카드 선택 UI | 주거 형태·활동량·하루 혼자 있는 시간·거주 공간·선호 크기를 카드 선택형으로 입력 | US-03 |
| 진행 상태 표시 | 현재 문항 / 전체 문항 수 표시 (예: 3 / 5) | US-03 |

| 비회원 횟수 초과 처리 | 2회 초과 시 회원가입 유도 모달 표시, 설문 진행 불가 | US-04 |

## 7. Out of Scope

| 항목 | 제외 이유 |
|---|---|
| 지도 기반 보호소 뷰 | 지도 API 비용·복잡도가 4주 개인 프로젝트 MVP 범위를 초과 |
| 커뮤니티(게시판, 댓글) | 핵심 가치(매칭·체크리스트)와 무관, v2에서 고려 |
| 산책 알림 (기상청 API 연동) | MVP 외 기능, 이번 개인 프로젝트 범위 밖 |
| 실제 결제·입양 신청 처리 | 보호소별 절차 상이, 연락처 원클릭 연결로 대체 |
| 소셜 로그인 (OAuth) | MVP에서 이메일 인증으로 단순 시작, 추후 확장 가능 |
| 관리자 페이지 | 개인 프로젝트 범위 밖 |
